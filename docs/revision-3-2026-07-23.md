# Revision 3 — 23 July 2026 (dashboard + mobile)

## 1. Dashboard header removed

`MarketingHeader` is gone from `auth-guard-shell.tsx`. The dashboard is now its
own environment: sidebar and content only on desktop. On mobile a compact bar
carries just the logo and a hamburger, matching the mobile mockups — the
hamburger opens the marketing links plus sign out, since those are no longer
reachable any other way from inside the dashboard.

## 2. Dashboard locked to the viewport

The shell is `h-screen` with `overflow-hidden`, so the frame always matches the
screen exactly as the mockups show — sidebar and mobile tab bar stay pinned.

**One deliberate deviation, worth knowing about.** The content well itself is
`overflow-y-auto` rather than clipped. A dashboard's content is data-driven — a
musician with eight pending requests, or a facility with a long announcement
list, produces more content than a short viewport can hold. Hard-clipping would
make those rows permanently unreachable, and on a laptop at 768px tall with
browser zoom the panels would be cut mid-row. So the *page* doesn't scroll, but
a full panel can scroll within its own area. If you'd rather have it clip
absolutely, that's a one-line change — say the word.

## 3. Sidebar icons fixed

The previous icons were hand-drawn SVG approximations, and several were plain
wrong — Availability rendered as a bare filled circle, Resources as a solid
block.

They now come from the design pack. `extract_nav_icons.py` reads each exported
sidebar row (the pack ships icon and label baked into one image), detects the
gap between the glyph and the text, crops out just the glyph, and writes it
square and centred to `public/mmm/nav/`. Nineteen icons extracted and
sanity-checked for blanks and solid blobs. Labels are still set live in Poppins,
so they stay selectable and translatable.

## 4. Mobile

- **Hero copy now overlays the banner at every breakpoint**, matching the mobile
  mockups, with type stepping down so it clears the photo. Verified at 390px and
  1440px: the headline sits inside the banner on all four pages, with no
  horizontal overflow.
- **Dashboards** get a fixed bottom tab bar on mobile instead of a drawer, using
  the same design-pack icons.
- **How It Works** keeps both columns side by side on mobile, per the mockup,
  with the card padding, circle sizes, and step type scaled down to fit a phone.
- **About**: Our Story becomes a vertical timeline with a dotted vertical
  connector on mobile and stays horizontal from `lg` up. Mission / Vision /
  Future Goals become three separate cards on mobile, one panel on desktop.
- **Contact**: E-mail Us and Call us sit side by side on mobile as in the
  mockup; the address wraps rather than overflowing.
- **Facility registration**: step 3 field order corrected to Phone → Preferred
  Contact Method → Job Title, and step 5's review is now the bordered
  label/value table the mockup shows.

## 5. Note on Step 1 of registration

predates Michael's request to split first and last name, so the split is kept —
the client request wins over the older mockup. Flagging it so it doesn't read as
an oversight.
