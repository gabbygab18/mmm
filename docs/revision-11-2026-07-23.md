**4 — Registration retention.** Root cause was the `on_auth_user_created`
trigger writing only `id`, `role` and `email` into `public.users`; every other
answer stayed in `auth.users.raw_user_meta_data` and never reached `musicians` /
`centers` / `center_locations`. Fixed by
`20260723090000_registration_retention_and_facility_fields.sql`, which rewrites
the trigger and backfills existing accounts from metadata already on file.

**This is not live until the migration is applied to the Supabase project.**
Until then new registrations still lose their answers. It's idempotent and safe
to re-run.

**5 — Human verification.** Works today with a built-in arithmetic challenge
plus a honeypot field and a minimum time-on-form check — verified working, since
the automated run above had to solve the challenge to reach Step 2. For
production-grade bot resistance, set `NEXT_PUBLIC_TURNSTILE_SITE_KEY` and
`TURNSTILE_SECRET_KEY` and it switches to Cloudflare Turnstile automatically; no
code change needed.

So: three are fully closed, one needs a database migration run, one works now
and gets stronger with keys.
