# WordPress Option Summary (Client-Facing)

Date: June 2, 2026

## Purpose
This summary explains what the platform would look like if built entirely in WordPress, what that means for day-to-day use, and what tradeoffs to expect.

## Short Answer
Yes, this can be built in WordPress.

WordPress is a strong choice if the top priority is easy content updates and handoff to non-technical teams.

For this project, WordPress can handle the required features, but some of the more advanced workflow pieces will still require custom development work behind the scenes.

## What You Would Get with WordPress

1. Easy content management
- Team members can update homepage text, educational content, and resources without developer help.

2. User accounts and profiles
- Musicians and centers can have accounts and profile pages.

3. Search and directory views
- Visitors can browse and search listings.

4. Calendar and map experiences
- Calendar pages and map pages can be added using WordPress tools and add-ons.

5. Admin moderation
- Admins can review and approve profiles and media submissions.

6. Scheduling and request workflow
- Yes, this can be included in a WordPress-only build.
- Musicians and centers can submit availability/request dates, send requests, and track status changes.
- Admins can intervene when needed.
- Important note: this is typically implemented with multiple add-ons plus custom workflow setup, not one out-of-the-box plugin.

## Where WordPress Is Less Ideal

1. Complex workflows
- Multi-step scheduling and approval flows are possible, but they usually require several add-ons plus custom setup.

2. Advanced filtering at scale
- Region + center + musician combinations and holistic calendar filtering can require custom optimization.

3. Ongoing maintenance
- WordPress plugin updates are normal, but they can sometimes cause compatibility issues that need troubleshooting.

## Cost Snapshot (Typical)

Software/add-on licensing (not including custom development labor):

- Lean setup: about 500 to 1,200 USD per year
- More robust setup: about 1,500 to 3,500+ USD per year

Additional costs may include:
- Map API usage (if needed)
- Background check or identity vendors (if used)
- Ongoing maintenance/support time

## How the Annual Cost Range Was Estimated

The ranges above come from combining common annual license costs for a typical WordPress stack for this kind of platform.

### Lean Setup Example (about 500 to 1,200 USD/year)
Typical assumptions:
- Membership/roles: free or low-cost tier
- Directory/search: free core + 1 to 2 paid addons
- Forms: paid basic plan

Illustrative annual total:

- Estimated total: about 500 to 1,200 per year

## Expected Add-Ons
A WordPress-only build would usually rely on add-ons for:

- Membership and roles
- Directory/search listings
- Application forms
- Workflow automation
- Calendar/events
- Maps
- Email delivery
- Video/gallery display
- Security hardening

Some add-ons are free with paid upgrades. Others are paid from the start.

## Practical Decision Guidance
Choose WordPress-only if your top priorities are:
- Easy handoff and content editing
- Faster setup for pages and listings
- Willingness to use add-ons and some custom development for advanced workflows

This path is viable. Just plan for custom work in the scheduling, moderation, and advanced filter areas so the experience stays reliable as usage grows.

## Recommended Next Step
If you choose WordPress-only, the next step is to finalize:

4. A phased launch plan (core launch first, advanced features next)
