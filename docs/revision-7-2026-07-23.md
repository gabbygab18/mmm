# Revision 7 — 23 July 2026 (wave position)

## What moved

Following the arrow on the Why Music Matters screenshot: the light-streak wave
now sits **on the hero/section boundary and overhangs into the section below**,
rather than floating inside the hero band.

Two changes made that possible.

**1. The wave is no longer clipped.** It was inside the band `div`, which
carries `overflow-hidden` for the photo crop — so it could never extend past the
bottom edge no matter what offset it was given. It now lives in the outer
`<section>`, outside the clip, with `z-10` so it paints over the following
section (which otherwise wins, being later in the document). The section got
`isolate` so that stacking context stays contained and can't leak.

**2. One offset now serves every page.** The wave core lands at

```
core_y = H − Ih × (0.32 − T)
```

Setting the core on the band's bottom edge means `f = 1.0`, and the `H` term
cancels — so the same `waveOffset` works regardless of band height. The
per-page overrides on FAQ and Contact are gone; the default is `36%`, which puts
the core about 42px past the boundary and gives roughly 370px of overhang.

pixels on Why Music Matters' navy band and 8.4% on About's.

No horizontal overflow at 390 / 768 / 1024 / 1440 across About, FAQ, Why Music
Matters, Contact, How It Works, Facility Registration and Home.

## Tuning

It's a single number now — `waveOffset` in `page-hero.tsx`, currently `36%`.
Larger pushes the wave further down and increases the overhang; `32%` sits the
core exactly on the boundary with no overhang. Any page can still override it
with the `waveOffset` prop if one hero wants a different treatment.
